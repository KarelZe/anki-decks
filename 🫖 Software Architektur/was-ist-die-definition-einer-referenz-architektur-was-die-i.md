## Front
Was ist die <b>Definition</b> einer <b>Referenz-Architektur</b>?
Was die Intuition einer <b>Referenz-Architektur</b>?

## Back
<div>
  <b>Definition:</b>
</div>
<div>
  Eine Referenzarchitektur ist eine Menge an grundsätzlichen
  Design-Entscheidungen, die simultan auf mehrere verwandte Systeme
  (typischerweise aus der selben Anwendungsdomäne) anwendbar sind,
  mit impliziten Variationspunkten wie z. B. dem Vorhandensein oder
  Abwesendsein einer Komponente.
</div>
<div>
  <b>Intuition:</b>
</div>
<div>
  Eine Referenz-Architektur ist damit ein Weg
  Software-Architekturen für spezifische Domains zu
  standardisieren.
</div>
<div>
  Eine Referenz-Architektur ist eine Vorlage für das Systemdesign.
</div>
<div>
  Nicht alle Komponenten einer Referenz-Architektur müssen bei der
  Definition eines spezifischen Systems genutzt werden.
</div>
<div>
  Jede Komponente einer spezifischen Architektur kann
  typischerweise auf eine Komponente der Referenz-Architektur
  gemappt werden.
</div>
