### Front
Ist es ausreichend nur die <b>Architektur</b> anzupassen, sofern
ein <b>change request</b> besteht?

### Back
Nein, es gibt noch mehr Tätigkeiten, die mit geändert werden
müssen: z. B. Buildskripte anpassen, Testfälle anpassen etc..
<div>
  Änderungsbedarf bei: Implementierung, Java Source Files,
  Building, Build config, Tests, Unit Tests für bereitgestellte
  Interfaces, Rollout, Deployment, Hibernate Configuration files
  für das OR Mapping.
</div>
<div>
  Die Anpassung der Architektur verursacht nur den geringsten
  Aufwand.
</div>
