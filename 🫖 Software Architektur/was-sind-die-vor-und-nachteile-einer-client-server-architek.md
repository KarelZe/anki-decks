## Front
Was sind die <b>Vor-</b> und <b>Nachteile</b> einer
<b>Client-Server-Architektur</b>

## Back
<p><strong>Vorteile</strong>
<ul>
  <li>Einfache Verteilung der Daten
  <li>Nutzt effektiv die Netzwerkstruktur aus
  <li>Es ist einfach bestehende Server hinzuzufügen oder upzugraden
</ul>
<p><strong>Nachteile</strong>
<ul>
  <li>kein gemeinsames Datenmodell der Sub-Systeme d. h. der
  Datenaustausch ist ggf. ineffizient
  <li>Redundantes Management der Server
  <li>Kein zentralisiertes Register mit Namen und Diensten
  (<em>Services</em>). Wodurch es ggf. schwierig ist, zu sehen,
  welche Server oder Dienste verfügbar sind.
</ul>
