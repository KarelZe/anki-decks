## Front
Worin unterscheiden sich <b>Architecture Patterns</b> von
<b>Architecture Styles</b>?

## Back
<p><strong>Architecture Patterns</strong>
<ul>
  <li>Konstruktive Vorgabe, wie Lösung aussehen soll
  <li>Konkreter
  <li>Expliziter
  <li>Rezept-basierend
  <li>Können kombiniert werden
</ul>
<p><strong>Architecture Styles</strong>
<ul>
  <li>Abstrakter
  <li>Geben größere Freiheiten bei der Realisierung
  <li>Legen <b>Beschränkungen</b> fest
  <li>Lassen eine konstruktive Beschreibung einer Lösung offen
  <li>Nur ein Architekturstil je Dimension (<em>decomposition</em>
  / <em>interaction</em>)
</ul>
