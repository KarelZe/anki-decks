### Front
Warum sollte man explizite Modelle für die Dokumentation von Software Architektur verwenden? Warum ist Code allein zum Beschreiben der Software Architektur nicht ausreichend.

### Back
<div>
  <div>
    <ul>
      <li>Code beschreibt nur ein unvollständiges Bild der
      Systemarchitektur. Programmiersprachen sind etwa nicht darauf
      ausgelegt um Designentscheidungen zu dokumentieren.
      <li>Ohne explizites Modell gibt es ggf. gar keine
      Architektur, da die Software mit der Zeit gewachsen ist. Die
      interne Struktur ist wegen dem noch lange keine Software
      Architektur! (Vgl. Steinhaufen Beispiel)
      <li>Eine explizite Architektur verbessert die Kommunikation
      zwischen Stakeholdern und Entwicklern.
      <li>Explizite Modelle erlauben eine Qualitätsvorhersage.
    </ul>
  </div>
</div>
